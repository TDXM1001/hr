二、技术选型明细（更新版）
层次	技术	用途
前端	Vue 3 + Vite + Element Plus	SPA 单页应用
前端状态	Pinia	全局状态管理
前端路由	Vue Router 4	动态路由 + 权限菜单
后端框架	Spring Boot 3.x（独立部署）	每个服务独立 Spring Boot 应用
反向代理	Nginx	路由分发 + 负载均衡 + 静态资源
服务间通信	OpenFeign（独立使用，无需 Spring Cloud）	服务间 HTTP 调用
配置管理	各服务独立 application.yml + 共享配置文件	无需配置中心
认证	Spring Security + JWT + Redis	无状态认证 + Token 共享
ORM	MyBatis-Plus	数据访问 + 数据权限插件
数据库	MySQL 8.0（按服务分库）	持久化存储
缓存	Redis	Token / 权限缓存 / 热数据
消息队列	RabbitMQ	异步事件（入职通知、考勤同步等）
文件存储	MinIO	员工证件、合同附件
接口文档	Knife4j (Swagger)	每个服务独立 API 文档
容器化	Docker + Docker Compose	本地开发 / 部署编排
三、7 个服务职责划分
服务	端口	核心职责	数据库
hr-gateway	—	Nginx 承担，无需独立服务	—
hr-auth	8081	登录/登出、Token 签发刷新、角色与权限管理、菜单管理	hr_auth
hr-org	8082	部门 CRUD、岗位管理、组织架构树、架构调整记录	hr_org
hr-employee	8083	员工档案 CRUD、合同管理、证件管理、批量导入导出	hr_employee
hr-attendance	8084	班次排班、打卡记录、假期额度、请假审批、月度考勤报表	hr_attendance
hr-transfer	8085	调岗/调薪/晋升申请、离职申请审批、交接记录、离职员工库	hr_transfer
hr-common	8086	数据字典、文件上传下载、消息通知、操作日志、审批流引擎	hr_common
四、服务间调用关系
hr-auth ◄──── 所有服务（校验 Token、获取当前用户权限）
    │
hr-org ◄──── hr-employee（查部门/岗位名称）
    │     ◄── hr-attendance（查部门下员工列表）
    │     ◄── hr-transfer（查岗位信息）
    │
hr-employee ◄── hr-attendance（查员工基础信息）
    │        ◄── hr-transfer（查员工当前岗位/部门）
    │
hr-common ◄── 所有服务（字典翻译、文件上传、消息推送）
通信方式：

同步调用：OpenFeign（HTTP REST），用于实时查询（如：查员工姓名）
异步事件：RabbitMQ，用于非实时通知（如：员工入职后通知考勤服务初始化排班）
五、数据权限模型
超级管理员 ──→ 全集团数据 (data_scope = ALL)
    │
HR 总监 ──→ 全集团数据 (data_scope = ALL，部分功能只读)
    │
分公司 HR ──→ 本公司数据 (data_scope = COMPANY)
    │
部门经理 ──→ 本部门数据 (data_scope = DEPARTMENT)
    │
普通员工 ──→ 仅个人数据 (data_scope = SELF)
实现方式： MyBatis-Plus 拦截器自动拼接 SQL 条件，根据当前用户的 data_scope 和所属组织 ID 过滤数据。

以上是更新后的完整架构设计。这个方案的核心变化：

✅ 用 Nginx 替代 Spring Cloud Gateway

✅ OpenFeign 独立使用，不依赖 Spring Cloud
✅ 无配置中心，各服务独立配置
